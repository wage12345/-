# Refactor Review

你正在为学生管理系统仓库执行重构任务。目标是提升可维护性，而不是引入新功能或大范围改写。

## 使用时机

- 用户要求“重构”“整理代码”“减少重复逻辑”“提高清晰度”
- 某个 controller / model / view 中存在重复分支或重复查询
- 需要在保持现有路由与页面行为不变的前提下优化实现

## 工作约束

1. 先阅读目标文件及其直接依赖，再决定是否抽取函数。
2. 保持路由路径、表单字段、数据库字段和页面输出不变，除非用户明确要求修改行为。
3. 优先复用现有目录结构：`controllers/`、`models/`、`routes/`、`views/`、`config/`。
4. 只在复杂逻辑前添加简短注释，不写解释性废话。
5. 任何重构都要补最小必要测试，至少覆盖改动触达的正常路径。

## 推荐流程

1. 用 `filesystem` 浏览目标模块、相关路由、测试和视图。
2. 识别重复代码、长函数、隐式耦合和不一致的数据整形方式。
3. 在最小边界内抽取辅助函数或收敛查询逻辑。
4. 运行 `npm test`，必要时补充 Jest + Supertest 用例。
5. 输出时说明“改了什么、为什么这样改、验证了什么”。
